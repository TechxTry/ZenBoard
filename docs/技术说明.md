# ZenBoard — 技术说明与开发参考

面向开发者与运维：技术栈、目录结构、本地开发、环境变量、ETL、容器与部署、HTTP API 摘要。

---

## 技术栈

| 层级 | 技术 |
|------|------|
| 后端 | Go 1.25、Gin、GORM（PostgreSQL + MySQL） |
| 前端 | React 18、TypeScript、Vite 6、Ant Design 5、React Router 7、Zustand |
| 容器 | 多阶段 Dockerfile（后端 Alpine 二进制；前端 Nginx 托管静态资源并反代 `/api`） |

## 仓库结构

```
zenboard/                    # 根目录名可与 Git 仓库名一致
├── backend/                 # Go API 与 ETL
│   ├── cmd/server/          # 入口
│   ├── internal/            # config、db、handlers、etl、scheduler、models …
│   ├── config/etl_tables.yaml
│   ├── migrations/001_init.sql
│   └── Dockerfile
├── frontend/                # React SPA
│   ├── src/
│   ├── nginx.conf           # 生产环境：SPA + /api → backend:8080
│   └── Dockerfile
└── docker-compose.yml       # 本地构建 backend / frontend 镜像并编排 PG + Redis
```

## 前置条件

- **本地开发（非 Docker）**：**PostgreSQL**（建议 15+）、Go、Node.js。
- **Docker 一键启动**：仅 **Docker**；编排会拉起 PostgreSQL 与 Redis。
- 可访问的 **禅道 MySQL**（只读即可；表名可能因禅道版本不同，需在 ETL 配置中调整）。同步前可在界面配置数据源。

Docker Compose 仍包含 **Redis** 服务（当前后端定时同步已改为进程内定时器，不依赖 Redis）。

## 本地开发

### 1. 初始化数据库

创建数据库与用户后，执行迁移：

```bash
psql "$PG_DSN" -f backend/migrations/001_init.sql
```

（`PG_DSN` 与仓库根目录 `.env.example` 中格式一致，或直接使用 `psql` 连接串。）

### 2. 后端

在仓库根目录准备环境变量（与 Docker Compose 共用同一文件）：

```bash
# 在仓库根目录
cp .env.example .env
# 编辑 .env：PG_DSN、REDIS_ADDR、ADMIN_*、JWT_SECRET 等

cd backend
go run ./cmd/server
```

默认监听 `PORT`（未设置时为 `8080`）。

### 3. 前端

```bash
cd frontend
npm install
npm run dev
```

开发服务器默认 **5173**，并通过 Vite 将 `/api` 代理到 `http://localhost:8080`。

### 4. 登录

使用 `.env` 中的 `ADMIN_USER` / `ADMIN_PASS` 在 `/login` 登录。

## 环境变量

完整说明见仓库根目录 [`.env.example`](../.env.example)。常用项：

| 变量 | 说明 |
|------|------|
| `PORT` | HTTP 端口，默认 `8080` |
| `WEB_PORT` | 前端对外端口（宿主机端口 → 容器 Nginx 80），默认 `2024` |
| `PG_DSN` | PostgreSQL 连接串 |
| `REDIS_ADDR` | Redis 地址，如 `localhost:6379` |
| `ADMIN_USER` / `ADMIN_PASS` | 管理员登录 |
| `JWT_SECRET` | JWT 签名密钥 |
| `ZT_HOST` 等 | 可选；禅道 MySQL，也可仅在前端「系统配置」填写 |
| `ETL_CONFIG_PATH` | ETL 映射文件路径，默认 `config/etl_tables.yaml`（相对后端工作目录） |

修改 `config/etl_tables.yaml` 后需**重启后端**。

## ETL 配置

[`backend/config/etl_tables.yaml`](../backend/config/etl_tables.yaml) 定义源表（禅道 MySQL）到目标表（PG `local_*`）的映射，以及水印类型：`none`（全量覆盖）、`time`、`id`。部分禅道版本表名不同（例如工时表），请按实际库表调整 `source` 与 `extra_filter`。

## Docker 与部署要点

- **编排** [`docker-compose.yml`](../docker-compose.yml)：从源码 **构建** `zenboard-backend:local` / `zenboard-frontend:local`，适合公开仓库 clone 后直接运行。
- 为减少“端口已被占用”类启动问题，Compose 中 **PostgreSQL / Redis / backend 默认不发布到宿主机端口**（仅容器网络内可访问）；对外只发布前端端口（默认 `2024`，可通过 `WEB_PORT` 覆盖）。
- **后端镜像**：工作目录 `/app`，镜像内已含 `config/etl_tables.yaml`；Compose 通过环境变量注入 `PG_DSN`、`REDIS_ADDR` 等。
- **前端镜像**：Nginx 将 `/api` 转发到 **`http://backend:8080`**，编排中服务名为 `backend`（见 [`frontend/nginx.conf`](../frontend/nginx.conf) 中的 `proxy_pass`）。

单独构建示例（在项目根目录）：

```bash
docker build -f backend/Dockerfile -t zenboard-backend:latest ./backend
docker build -f frontend/Dockerfile -t zenboard-frontend:latest ./frontend
```

生产环境若使用私有镜像仓库，可自行编写 `docker compose` 编排，将 `image` 指向仓库地址并配置 `docker login`，不在本仓库维护。

## HTTP API（摘要）

- `POST /api/login` — 登录
- `GET/PUT /api/config/datasource`、`POST /api/config/datasource/test` — 数据源（需 JWT）
- `GET/PUT /api/config/sync-settings` — 自动同步周期间隔（分钟，需 JWT）
- `GET /api/users`、`/api/groups` 相关 — 用户与项目组
- `GET /api/workbench/*` — 工作台列表
- `POST /api/sync/trigger`、`GET /api/sync/status` — 同步

具体路由定义见 [`backend/cmd/server/main.go`](../backend/cmd/server/main.go)。
